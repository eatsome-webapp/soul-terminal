# Plan 06: main.dart herschrijven + AndroidManifest merge

<objective>
De flutter_module main.dart herschrijven met SOUL app init-flow (ProviderScope, ObjectBox, API key, Sentry) als FlutterFragment-compatible root widget, de Pigeon bridge setup behouden, en de AndroidManifest mergen met SOUL permissions en services.
</objective>

<must_haves>
- main.dart bevat UncontrolledProviderScope als root widget wrapping
- main.dart volgt de SOUL init-volgorde: ObjectBox → ProviderContainer → API key → OpenClaw → Notifications → ForegroundService → Sentry → widget build
- SoulBridgeApi.setUp() wordt aangeroepen in het nieuwe root widget (niet verloren)
- AndroidManifest bevat READ_CONTACTS en READ_CALENDAR permissions
- AndroidManifest bevat SOUL ForegroundService en NotificationListenerService
- CI build groen na push
</must_haves>

<task id="06.1">
<title>Herschrijf main.dart met SOUL init-flow</title>
<read_first>
- ~/soul-app/apps/soul/lib/main.dart — de broncode init-flow (VERPLICHT lezen)
- flutter_module/lib/main.dart — huidige BridgeTestScreen (wordt vervangen)
- flutter_module/lib/core/di/providers.dart — provider definities
- flutter_module/lib/generated/terminal_bridge.g.dart — SoulBridgeApi interface
</read_first>
<action>
Herschrijf flutter_module/lib/main.dart. De nieuwe main.dart moet:

1. **`main()` functie:**
   - `WidgetsFlutterBinding.ensureInitialized()`
   - `runApp(UncontrolledProviderScope(container: container, child: SoulInitWidget()))`
   - SoulInitWidget toont een loading screen terwijl async init draait

2. **SoulInitWidget (StatefulWidget):**
   - In `initState()`: start async init (ObjectBox, API key, OpenClaw, etc.)
   - Toont loading indicator tijdens init
   - Na init: navigeert naar SoulApp (met GoRouter)
   - Implementeert `SoulBridgeApi` en roept `SoulBridgeApi.setUp(this)` aan in `initState()`
   - In `dispose()`: `SoulBridgeApi.setUp(null)`

3. **Async init-flow (EXACT de volgorde uit SOUL main.dart):**
   ```
   ObjectBox openStore() → store
   ProviderContainer met objectBoxStoreProvider.overrideWithValue(store)
   API key laden → apiKeyNotifierProvider.notifier.setKey()
   OpenClaw client init (try/catch, mag falen)
   LocalNotificationService.initialize() (try/catch)
   ForegroundServiceManager.initialize() + startService() (try/catch)
   Initial route bepalen (projectDao check)
   SentryConfig.init() wrapping
   ```

4. **SoulBridgeApi implementatie:**
   De huidige BridgeTestScreen implementeert SoulBridgeApi met `onTerminalOutput` en `onSessionChanged`. Deze callbacks moeten behouden blijven in SoulInitWidget of een apart widget.

5. **SoulApp widget:** Behoud uit SOUL main.dart (DynamicColorBuilder + MaterialApp.router).

KRITIEK: Lees EERST de broncode main.dart uit soul-app. Neem niet aan — kopieer de exacte init-flow.
</action>

<task id="06.3">
<title>Merge AndroidManifest.xml met SOUL permissions en services</title>
<read_first>
- app/src/main/AndroidManifest.xml — huidige soul-terminal manifest
- ~/soul-app/apps/soul/android/app/src/main/AndroidManifest.xml — SOUL manifest
</read_first>
<action>
Voeg de ontbrekende SOUL permissions en services toe aan `app/src/main/AndroidManifest.xml`:

**Permissions toevoegen (na bestaande uses-permission regels):**
```xml
<uses-permission android:name="android.permission.READ_CONTACTS" />
<uses-permission android:name="android.permission.READ_CALENDAR" />
```

Opmerking: FOREGROUND_SERVICE, FOREGROUND_SERVICE_SPECIAL_USE, RECEIVE_BOOT_COMPLETED, POST_NOTIFICATIONS, WAKE_LOCK, MANAGE_EXTERNAL_STORAGE zijn al aanwezig. Alleen READ_CONTACTS en READ_CALENDAR ontbreken.

**Services toevoegen (in <application>, na bestaande services):**
```xml
<!-- SOUL foreground service (flutter_foreground_task plugin) -->
<service
    android:name="com.pravera.flutter_foreground_task.service.ForegroundService"
    android:foregroundServiceType="specialUse"
    android:exported="false">
    <property
        android:name="android.app.PROPERTY_SPECIAL_USE_FGS_SUBTYPE"
        android:value="AI assistant providing proactive briefings and contextual monitoring" />
</service>

<!-- SOUL notification listener (notification_listener_service plugin) -->
<service
    android:label="SOUL Notifications"
    android:name="notification.listener.service.NotificationListener"
    android:permission="android.permission.BIND_NOTIFICATION_LISTENER_SERVICE"
    android:exported="true">
    <intent-filter>
        <action android:name="android.service.notification.NotificationListenerService" />
    </intent-filter>
</service>
```

KRITIEK: TermuxService, RunCommandService, en alle bestaande entries ONGEWIJZIGD laten.
</action>
