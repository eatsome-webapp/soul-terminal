<action>
In `flutter_module/lib/core/di/providers.dart`, change `objectBoxStoreProvider` from the current pattern (throws `UnimplementedError` unless overridden) to a `StateProvider<Store?>` that starts as `null`:

Replace:
```dart
final objectBoxStoreProvider = Provider<Store>((ref) {
  throw UnimplementedError('objectBoxStoreProvider must be overridden');
});
```

With:
```dart
final objectBoxStoreProvider = StateProvider<Store?>((ref) => null);
```

This allows main.dart to set the store after async init without needing `ProviderContainer` overrides. All consumers that use `ref.watch(objectBoxStoreProvider)` will need to handle the nullable return â€” but they are guarded by the loading screen in main.dart (task 06-03-02) which blocks widget rendering until store is non-null.

Also check all direct usages of `objectBoxStoreProvider` in the codebase (likely in DAOs and services). If they use `ref.watch(objectBoxStoreProvider)` expecting a non-null `Store`, they will get `Store?`. Add a convenience provider:

```dart
/// Non-null store provider â€” only use after initialization is confirmed complete.
/// Throws if accessed before store is ready (guarded by SoulInitWidget).
final objectBoxStoreReadyProvider = Provider<Store>((ref) {
  final store = ref.watch(objectBoxStoreProvider);
  if (store == null) throw StateError('ObjectBox store not yet initialized');
  return store;
});
```

Update all existing references from `objectBoxStoreProvider` to `objectBoxStoreReadyProvider` in provider definitions that need a non-null Store (e.g., vectorStoreProvider, memoryServiceProvider).
</action>
