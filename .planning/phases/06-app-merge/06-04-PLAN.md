# Plan 04: Database, Service & API Key Validation

<objective>
Validate and ensure that Drift database paths, ObjectBox store paths, foreground service configuration, and API key storage all work correctly in the `com.soul.terminal` FlutterFragment context. Fix any hardcoded paths or configuration that assumes standalone SOUL app context. Verify notification channel IDs and notification IDs do not conflict with Termux's native services.
</objective>

<must_haves>
- Drift SQLite database writes to `/data/data/com.soul.terminal/files/` directory (via path_provider)
- ObjectBox store writes to `/data/data/com.soul.terminal/files/` directory (via path_provider)
- Neither database path overlaps with Termux's `$HOME` at `/data/data/com.soul.terminal/files/home/`
- SOUL ForegroundService uses channel ID `soul_foreground` (not conflicting with Termux)
- SOUL notification ID is `256` (not conflicting with Termux's IDs)
- `flutter_secure_storage` / ApiKeyService works under `com.soul.terminal` package context
- No hardcoded paths referencing `com.soul.app` or other non-`com.soul.terminal` package names
</must_haves>

Read `flutter_module/lib/services/database/soul_database.dart` and verify the `_openConnection()` method.

Expected pattern:
```dart
static QueryExecutor _openConnection() {
  return LazyDatabase(() async {
    final dbFolder = await getApplicationDocumentsDirectory();
    final file = File(p.join(dbFolder.path, 'soul.sqlite'));
    return NativeDatabase.createInBackground(file);
  });
}
```

`getApplicationDocumentsDirectory()` in FlutterFragment context returns the host app's documents directory: `/data/data/com.soul.terminal/files/Documents/` or `/data/data/com.soul.terminal/app_flutter/`. This is NOT under Termux's `$HOME` (`/data/data/com.soul.terminal/files/home/`), so no conflict.

If the code uses `getApplicationDocumentsDirectory()` → no changes needed.
If the code hardcodes any path with a package name → change to use `path_provider`.
If the code uses a custom directory name → verify it doesn't conflict with Termux's directories.

Also verify all `.drift` schema files are present in `flutter_module/lib/services/database/tables/`. List them and count — should be ~25 `.drift` files.
</action>

- No hardcoded `/data/data/` paths in `soul_database.dart`

```dart
    await _storage.write(key: 'anthropic_api_key', value: key);
  }
}
```

If the storage uses default options → no changes needed (Android Keystore is per-package).
If it uses `AndroidOptions(encryptedSharedPreferences: true)` → fine, works per-package.
If it references `com.soul.app` anywhere → change to remove that reference.
