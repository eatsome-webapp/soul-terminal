```dart
class CommandWhitelist {
  // Check structured command against destructive patterns.
  // Returns true if command is destructive and needs confirmation.
  static bool isDestructive(String executable, List<String> args) { ... }

  // Also check combined string for bash -c bypass detection.
  // Returns human-readable description of why it's blocked.
  static String? getDestructiveReason(String executable, List<String> args) { ... }
}
```

Destructive patterns to implement (match case-insensitive on executable, exact on flags):
1. executable=`rm` AND args contains `-rf` or `-r` combined with `-f` (any order, also `-Rf`, `--recursive --force`)
2. executable=`dd` AND any arg starts with `if=`
3. executable=`mkfs` (any args)
4. executable=`git` AND args contains `reset` AND `--hard`
5. executable=`chmod` AND args contains `-R` AND `777`
6. executable=`bash` or `sh` or `zsh` AND args contains `-c` — then scan the command string arg for patterns 1-5 plus `:(){:|:&};:` and `> /dev/sda`
7. Fork bomb pattern: any arg contains `(){` and `|` and `&}`

`isDestructive()` returns `true` if any pattern matches.
`getDestructiveReason()` returns a Dutch description string like `"rm -rf verwijdert bestanden recursief en geforceerd"` or `null` if safe.
</action>

Create `flutter_module/lib/services/awareness/ansi_stripper.dart`:

```dart
class AnsiStripper {
  /// Regex that matches ANSI escape sequences:
  /// - CSI sequences: \x1B[ followed by params and final byte
  /// - OSC sequences: \x1B] to BEL or ST
  /// - Single-char escapes: \x1B followed by @-Z or \-_
  /// - Also strips bare control characters (except \n, \r, \t)
  static final _ansiPattern = RegExp(
    r'\x1B(?:[@-Z\\-_]|\[[0-?]*[ -/]*[@-~]|\][^\x07]*(?:\x07|\x1B\\))',
  );

  static final _controlChars = RegExp(
    r'[\x00-\x08\x0B\x0C\x0E-\x1A\x1C-\x1F\x7F]',
  );

  /// Strip ANSI escape codes and control characters from terminal output.
  /// Preserves newlines (\n), carriage returns (\r), and tabs (\t).
  static String strip(String input) {
    return input
        .replaceAll(_ansiPattern, '')
        .replaceAll(_controlChars, '');
  }
}
```

The regex covers:
- CSI sequences like `\x1B[31m` (color), `\x1B[2J` (clear screen), `\x1B[?25h` (cursor)
- OSC sequences like `\x1B]0;title\x07` (set title)
- Single-character escapes like `\x1BM` (reverse line feed)
- Bracketed paste markers `\x1B[200~` and `\x1B[201~`
- Control characters that leak through (except whitespace)
</action>
