# Plan 05: UI layer kopiëren + build_runner codegen compleet

<objective>
Alle 64 UI source files kopiëren naar flutter_module (chat, conversations, settings, shell, etc.) en build_runner draaien om alle gegenereerde files voor het hele project te produceren — Drift, Freezed, ObjectBox, Riverpod, JsonSerializable.
</objective>

<must_haves>
- Alle 64 UI source files gekopieerd (NIET .g.dart of .freezed.dart)
- Alle UI subdirectories aanwezig: agentic, chat, common, conversations, decisions, onboarding, profile, projects, settings, setup, shell, terminal, vessels
- build_runner genereert alle .g.dart en .freezed.dart files succesvol (0 errors)
- Geen `package:soul_app/` imports in enige UI file
- Pigeon generated files (terminal_bridge.g.dart, system_bridge.g.dart) ongewijzigd behouden
</must_haves>

<task id="05.3">
<title>Draai build_runner voor volledig project</title>
<read_first>
- flutter_module/pubspec.yaml (dev_dependencies met build_runner, drift_dev, freezed, etc.)
</read_first>
<action>
Nu alle source files aanwezig zijn, draai build_runner voor het complete project:

```bash
cd flutter_module
flutter pub get
dart run build_runner build --delete-conflicting-outputs
```

Dit genereert:
- `objectbox.g.dart` in lib/ (van objectbox-model.json + @Entity models)
- `soul_database.g.dart` (Drift schema + DAOs)
- `*.freezed.dart` voor alle @freezed models
- `*.g.dart` voor @JsonSerializable, @riverpod annotaties
- DAO .g.dart files

Verwacht: build_runner exit code 0, geen errors.

Als er errors zijn:
1. Missing imports → fix imports
2. Conflicting generated files → gebruik --delete-conflicting-outputs (al in commando)
3. ObjectBox model mismatch → verifieer objectbox-model.json is correct gekopieerd
</action>
<acceptance_criteria>
- `flutter_module/lib/objectbox.g.dart` bestaat (gegenereerd)
- `flutter_module/lib/services/database/soul_database.g.dart` bestaat (gegenereerd)
- build_runner exit code 0
- CI build MOET groen zijn na push (main.dart is nog BridgeTestScreen, nieuwe files worden niet geïmporteerd)
</acceptance_criteria>
</task>
