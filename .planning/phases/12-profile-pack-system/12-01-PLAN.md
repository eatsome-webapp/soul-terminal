# Plan 12-01: GitHub Actions Profile Pack Build Workflow

<objective>
Create a GitHub Actions workflow that builds profile packs (pre-built zip of $PREFIX delta) for any profile defined in manifest.json, publishes them as GitHub Release assets with SHA-256 checksums, and auto-updates manifest.json with real URLs/sha256/sizeBytes. This enables Tier 1 fast-path installation (<60s) and makes the profile system pluggable for community contributors.
</objective>

<task id="1">
<title>Create profile pack build script</title>
<read_first>
- .github/workflows/bootstrap_build.yml (pattern to follow for Docker + checksum + release)
- profile-packs/manifest.json (current schema with empty URLs)
- app/src/main/java/com/termux/app/ProfilePackInstaller.java (expected zip format: SYMLINKS.txt + files relative to $PREFIX)
</read_first>
<action>
Create `profile-packs/build-profile-pack.sh` — a bash script that runs INSIDE the Docker container to build one profile pack.

```bash
#!/bin/bash
set -euo pipefail

# Arguments
PROFILE_ID="${1:?Usage: build-profile-pack.sh <profile-id>}"
VERSION_TAG="${2:?Usage: build-profile-pack.sh <profile-id> <version-tag>}"

# Paths
STAGING_PREFIX="/data/data/com.soul.terminal/files/usr"
BASE_SNAPSHOT="/tmp/base-snapshot.txt"
OUTPUT_DIR="/output"

echo "=== Building profile pack: $PROFILE_ID version $VERSION_TAG ==="

# Step 1: Snapshot base $PREFIX state (file listing with timestamps)
find "$STAGING_PREFIX" -type f -o -type l | sort > "$BASE_SNAPSHOT"
touch /tmp/timestamp-marker

# Step 2: Install packages based on profile
case "$PROFILE_ID" in
  claude-code)
    apt-get update
    apt-get install -y nodejs git gh
    npm install -g @anthropic-ai/claude-code
    ;;
  python)
    apt-get update
    apt-get install -y python python-pip git
    ;;
  *)
    echo "ERROR: Unknown profile: $PROFILE_ID"
    exit 1
    ;;
esac

# Step 3: Find new/modified files (delta)
DELTA_DIR="/tmp/profile-delta"
mkdir -p "$DELTA_DIR"

# Files newer than our timestamp marker
find "$STAGING_PREFIX" -newer /tmp/timestamp-marker -type f | while IFS= read -r filepath; do
  rel="${filepath#$STAGING_PREFIX/}"
  mkdir -p "$DELTA_DIR/$(dirname "$rel")"
  cp "$filepath" "$DELTA_DIR/$rel"
done

# Step 4: Build SYMLINKS.txt from new symlinks
find "$STAGING_PREFIX" -newer /tmp/timestamp-marker -type l | while IFS= read -r lnk; do
  target=$(readlink "$lnk")
  rel="${lnk#$STAGING_PREFIX/}"
  echo "${target}←${rel}"
done > "$DELTA_DIR/SYMLINKS.txt"

# Handle case where no new symlinks exist (empty file is fine)

# Step 5: Create zip
PACK_FILENAME="profile-pack-${PROFILE_ID}-aarch64.zip"
cd "$DELTA_DIR"
zip -r "$OUTPUT_DIR/$PACK_FILENAME" . -x "*.pyc"

# Step 6: Strip unnecessary files to reduce size
# Remove docs, man pages, __pycache__, .map files from the zip
zip -d "$OUTPUT_DIR/$PACK_FILENAME" "share/doc/*" "share/man/*" "**/__pycache__/*" "**.map" 2>/dev/null || true

# Step 7: Compute SHA-256
SHA256=$(sha256sum "$OUTPUT_DIR/$PACK_FILENAME" | cut -d' ' -f1)
SIZE_BYTES=$(stat -c%s "$OUTPUT_DIR/$PACK_FILENAME")

echo "=== Build complete ==="
echo "FILE: $PACK_FILENAME"
echo "SHA256: $SHA256"
echo "SIZE: $SIZE_BYTES bytes"

# Write metadata for workflow consumption
cat > "$OUTPUT_DIR/metadata.json" << EOF
{
  "profile_id": "$PROFILE_ID",
  "version": "$VERSION_TAG",
  "sha256": "$SHA256",
  "size_bytes": $SIZE_BYTES,
  "filename": "$PACK_FILENAME"
}
EOF
```

Make the script executable: `chmod +x profile-packs/build-profile-pack.sh`
</action>

<task id="2">
<title>Create GitHub Actions workflow for profile pack builds</title>
<read_first>
- .github/workflows/bootstrap_build.yml (Docker pattern, QEMU, release creation, checksum step)
- profile-packs/build-profile-pack.sh (script from task 1)
</read_first>
<action>
Create `.github/workflows/profile_pack_build.yml`:

```yaml
name: Profile Pack Build

on:
  workflow_dispatch:
    inputs:
      profile_id:
        description: "Profile to build"
        required: true
        type: choice
        options:
          - claude-code
          - python
      version_tag:
        description: "Version tag (e.g., 2026.03.22-r1)"
        required: true
        type: string

jobs:
  build-profile-pack:
    runs-on: ubuntu-24.04
    timeout-minutes: 360

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up QEMU for aarch64 emulation
        uses: docker/setup-qemu-action@v3
        with:
          platforms: arm64

      - name: Enable FUSE on host
        run: |
          sudo apt-get install -y fuse3
          sudo modprobe fuse || true

      - name: Download base bootstrap
        run: |
          # Get latest bootstrap release
          BOOTSTRAP_TAG=$(gh release list --repo ${{ github.repository }} --limit 20 | grep 'bootstrap-' | head -1 | awk '{print $1}')
          if [ -z "$BOOTSTRAP_TAG" ]; then
            echo "ERROR: No bootstrap release found"
            exit 1
          fi
          echo "Using bootstrap: $BOOTSTRAP_TAG"
          gh release download "$BOOTSTRAP_TAG" --repo ${{ github.repository }} --pattern "bootstrap-aarch64.zip" --dir /tmp
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

      - name: Build profile pack in Docker
        run: |
          mkdir -p output

          docker run --rm --privileged \
            --platform linux/arm64 \
            --device /dev/fuse \
            -v "$PWD:/workspace" \
            -v "$PWD/output:/output" \
            -v "/tmp/bootstrap-aarch64.zip:/tmp/bootstrap.zip" \
            ghcr.io/termux/package-builder \
            bash -c "
              set -euo pipefail

              # Extract base bootstrap into staging $PREFIX
              STAGING_PREFIX='/data/data/com.soul.terminal/files/usr'
              mkdir -p \$STAGING_PREFIX

              # Bootstrap is a zip of files relative to SYMLINKS.txt root
              cd /tmp && unzip -q bootstrap.zip -d bootstrap-staging
              # Move extracted content to staging prefix
              cp -a /tmp/bootstrap-staging/* \$STAGING_PREFIX/ 2>/dev/null || true

              # Process SYMLINKS.txt from bootstrap
              if [ -f /tmp/bootstrap-staging/SYMLINKS.txt ]; then
                while IFS='←' read -r target linkpath; do
                  linkfull=\"\$STAGING_PREFIX/\$linkpath\"
                  mkdir -p \$(dirname \"\$linkfull\")
                  ln -sf \"\$target\" \"\$linkfull\" 2>/dev/null || true
                done < /tmp/bootstrap-staging/SYMLINKS.txt
              fi

              # Set up apt sources for Termux packages
              mkdir -p \$STAGING_PREFIX/etc/apt/sources.list.d
              echo 'deb https://packages.termux.dev/apt/termux-main stable main' > \$STAGING_PREFIX/etc/apt/sources.list

              # Build the profile pack
              chmod +x /workspace/profile-packs/build-profile-pack.sh
              /workspace/profile-packs/build-profile-pack.sh '${{ inputs.profile_id }}' '${{ inputs.version_tag }}'
            "

      - name: Read build metadata
        id: meta
        run: |
          cat output/metadata.json
          SHA256=$(jq -r '.sha256' output/metadata.json)
          SIZE_BYTES=$(jq -r '.size_bytes' output/metadata.json)
          FILENAME=$(jq -r '.filename' output/metadata.json)
          echo "sha256=$SHA256" >> "$GITHUB_OUTPUT"
          echo "size_bytes=$SIZE_BYTES" >> "$GITHUB_OUTPUT"
          echo "filename=$FILENAME" >> "$GITHUB_OUTPUT"

      - name: Create GitHub Release
        uses: softprops/action-gh-release@v2
        with:
          tag_name: "profile-pack-${{ inputs.profile_id }}-${{ inputs.version_tag }}"
          name: "Profile Pack: ${{ inputs.profile_id }} ${{ inputs.version_tag }}"
          body: |
            Profile pack voor SOUL Terminal
            Profiel: ${{ inputs.profile_id }}
            Versie: ${{ inputs.version_tag }}
            Architectuur: aarch64
            Grootte: ${{ steps.meta.outputs.size_bytes }} bytes
            SHA-256: ${{ steps.meta.outputs.sha256 }}
          files: |
            output/${{ steps.meta.outputs.filename }}
          fail_on_unmatched_files: true
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}

      - name: Update manifest.json
        run: |
          DOWNLOAD_URL="https://github.com/${{ github.repository }}/releases/download/profile-pack-${{ inputs.profile_id }}-${{ inputs.version_tag }}/${{ steps.meta.outputs.filename }}"

          # Update manifest.json using jq
          jq --arg id "${{ inputs.profile_id }}" \
             --arg version "${{ inputs.version_tag }}" \
             --arg sha256 "${{ steps.meta.outputs.sha256 }}" \
             --argjson sizeBytes "${{ steps.meta.outputs.size_bytes }}" \
             --arg url "$DOWNLOAD_URL" \
             '(.profiles[] | select(.id == $id)) |= (.version = $version | .sha256 = $sha256 | .sizeBytes = $sizeBytes | .url = $url)' \
             profile-packs/manifest.json > /tmp/manifest-updated.json

          mv /tmp/manifest-updated.json profile-packs/manifest.json
          cat profile-packs/manifest.json

      - name: Commit updated manifest
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          git add profile-packs/manifest.json
          git diff --staged --quiet && echo "No manifest changes" || \
            git commit -m "ci: update manifest for ${{ inputs.profile_id }} ${{ inputs.version_tag }}" && \
            git push
```
</action>

<action>
Update `profile-packs/manifest.json` to schema version 2. Add a `maintainer` and `repository` field to each profile entry, enabling community-contributed profiles:

```json
{
  "schemaVersion": 2,
  "manifestUrl": "https://raw.githubusercontent.com/eatsome-webapp/soul-terminal/master/profile-packs/manifest.json",
  "profiles": [
    {
      "id": "claude-code",
      "name": "Claude Code",
      "description": "Node.js, Git, GitHub CLI, Claude Code CLI",
      "icon": "terminal",
      "version": "2026.03.22-r1",
      "minBootstrapVersion": "1",
      "arch": "aarch64",
      "sizeBytes": 0,
      "sha256": "",
      "url": "",
      "maintainer": "eatsome-webapp",
      "repository": "https://github.com/eatsome-webapp/soul-terminal"
    },
    {
      "id": "python",
      "name": "Python",
      "description": "Python, pip, Git",
      "icon": "code",
      "version": "2026.03.22-r1",
      "minBootstrapVersion": "1",
      "arch": "aarch64",
      "sizeBytes": 0,
      "sha256": "",
      "url": "",
      "maintainer": "eatsome-webapp",
      "repository": "https://github.com/eatsome-webapp/soul-terminal"
    }
  ]
}
```

Update `flutter_module/lib/services/profile_pack/profile_manifest.dart` to parse the new fields:

In `ProfileManifest.fromJson`, add:
```dart
final manifestUrl = json['manifestUrl'] as String? ?? '';
```

Add field `final String manifestUrl;` to `ProfileManifest` constructor.

In `ProfileEntry`, add two new fields:
```dart
final String maintainer;
final String repository;
```

In `ProfileEntry.fromJson`, add:
```dart
maintainer: json['maintainer'] as String? ?? '',
repository: json['repository'] as String? ?? '',
```

In `ProfileEntry` constructor, add the two required named parameters with these defaults.
</action>
