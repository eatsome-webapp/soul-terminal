**5. In the GestureAndScaleRecognizer.Listener anonymous class, in the `onScroll()` method (around line 170), wrap the scroll logic with nested scroll dispatching.**

Current `onScroll()` (approximately):
```java
public boolean onScroll(MotionEvent e, float distanceX, float distanceY) {
    if (mEmulator == null) return true;
    if (mEmulator.isMouseTrackingActive() && e.isFromSource(InputDevice.SOURCE_MOUSE)) {

    }
    scrolledWithFinger = true;
```

Replace the scroll dispatch portion (after `scrolledWithFinger = true;`) with:
```java
scrolledWithFinger = true;
distanceY += mScrollRemainder;
int deltaRows = (int) (distanceY / mRenderer.mFontLineSpacing);
mScrollRemainder = distanceY - deltaRows * mRenderer.mFontLineSpacing;
int dy = (int) distanceY;

// Let parent (BottomSheetBehavior) consume scroll first
if (dispatchNestedPreScroll(0, dy, mNestedScrollConsumed, mNestedScrollOffset, ViewCompat.TYPE_TOUCH)) {
    dy -= mNestedScrollConsumed[1];
    // Recalculate deltaRows based on remaining scroll
    deltaRows = (int) (dy / mRenderer.mFontLineSpacing);
}

if (deltaRows != 0) {
    doScroll(e, deltaRows);
}
dispatchNestedScroll(0, mNestedScrollConsumed[1], 0, dy - mNestedScrollConsumed[1], mNestedScrollOffset, ViewCompat.TYPE_TOUCH);
```

Actually, for BottomSheetBehavior, the simpler and more correct approach is: TerminalView should ALWAYS consume its own scroll and NOT let the parent intercept. The drag handle is the exclusive sheet drag zone. So the NestedScrollingChild3 implementation should signal to the parent "I'm scrolling, don't intercept me" by starting nested scroll but consuming everything itself.

Simplified approach — just start/stop nested scroll to prevent BottomSheetBehavior interception, but always consume scroll in TerminalView:

In the `onUp()` callback (around line 143):
After `mScrollRemainder = 0.0f;` add:
```java
stopNestedScroll(ViewCompat.TYPE_TOUCH);
```

In the `onScroll()` method, before the existing scroll logic, add:
```java
if (!scrolledWithFinger) {
    startNestedScroll(ViewCompat.SCROLL_AXIS_VERTICAL, ViewCompat.TYPE_TOUCH);
}
```

Keep the existing scroll logic unchanged — TerminalView always consumes its own scroll. The `startNestedScroll` call tells BottomSheetBehavior that a nested scroll is happening, which prevents it from intercepting the touch.
